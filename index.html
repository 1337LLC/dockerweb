Lazy Loading with STARGZ

The STARGZ Lazy Loading demo showcases high-performance container execution by enabling lazy loading of container layers. This approach is ideal for scenarios where:

  • You need to run arbitrary Docker images without pre-downloading the entire image (potentially gigabytes)
  • You want to minimize initial load time and only fetch needed layers on-demand
  • You want to leverage browser-native HTTP range requests for efficient streaming
  • You're building a platform to let users run containers from stargz-containers easily

Key Benefits:

  • Instant Startup: Begin running containers in milliseconds instead of waiting for multi-gigabyte downloads
  • Bandwidth Efficiency: Only download the layers and files that are actually used
  • On-Demand Streaming: Additional layers are fetched transparently as needed during execution
  • Scalability: Many users can load containers simultaneously without overwhelming servers

For implementation details and to build your own STARGZ-compatible images, see:

  • STARGZ Snapshotter - Lazy-loading snapshotter for containerd
  • stargz-containers - Pre-built STARGZ container images
  • container2wasm - Container to WebAssembly converter with STARGZ support

NOTE: Containers are compiled as WASI images and run on browser via polyfill. STARGZ-enabled demos support progressive layer loading via HTTP range requests.
